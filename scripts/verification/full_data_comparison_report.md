> Phase 2D の `wide ≈ umaren × 0.40` の推定係数は過大評価。実測では ワイド払戻は各ペアで大きく散らばり (1-2ペアは 0.3×umaren、1-3/2-3 ペアは 0.1×〜0.4×umaren)、平均で 0.4 を下回るケースが多い。

## 7. 未解決の課題 (Phase 2E で明らかになった)

### 7-1. Phase 2A (前走クラス) が past race では機能しない

- `race.netkeiba.com/race/shutuba.html` は **過去レースでは前走リンクを出力しない**
- 現在の `classifyPrevRace` は shutuba の前走リンクに依存
- **対応**: 近い将来のレース (未走) では機能するため、本番運用では有効
- **バックテスト時の拡張**: `db.netkeiba.com/horse/{id}/` から過去レース履歴を引いて補完する別パスが必要

### 7-2. Phase 2B (血統) の URL 構造が想定と異なる

- `db.netkeiba.com/horse/{id}/` には blood_table がない
- 実体は `db.netkeiba.com/horse/ped/{id}/`
- 種牡馬統計も `/horse/sire/{id}/` だが **距離帯別連対率が直接取れない**
- **対応**: Phase 2E では `DISABLE_SIRE=true` で無効化。別フェーズで selector を再設計

### 7-3. 枠連バックテストができない

- `predictions` に `waku` フィールドがないため戦略馬→枠マッピング不可
- **対応**: `collect-verification.ts` の `buildVerificationData` に waku を追加 → 次回再収集時から利用可
