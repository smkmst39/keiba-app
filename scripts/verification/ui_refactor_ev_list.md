# EV一覧 再構成（モバイル優先）実装報告

- 生成日時: 2026-04-23
- 修正ファイル: `app/components/RaceReport.tsx`
- 変更範囲: Section 6「全頭データサマリー」→「全頭EV一覧」

## 1. 現状分析

### 改修前の Section 6 (全頭データサマリー) の問題点

| 項目 | 状況 |
|---|---|
| 実装形式 | `<table>` with `minWidth: 600px` + `overflowX: 'auto'` |
| 列数 | **9列** (番・馬名・印・人気・オッズ・騎手・脚質・調教・総合スコア) |
| **EV 列** | **なし**（「EV 一覧」と呼ばれていたが、実は EV が表示されていなかった） |
| スコア順位 | 暗黙（表の並び順のみ、明示バッジなし） |
| モバイル表示 | 横スクロール発生、情報過密、視認性低 |
| EV 色分け | **適用されていない** |

### 改修前の表示専有率 (画面幅 390px)

- 表幅が 600px 以上のため、横スクロール操作が必須
- 縦方向も 1 行あたり `padding: 0.4rem 0.6rem` の 8〜16 行で全頭表示 → 画面をほぼ独占

## 2. 改修案の比較

| 案 | 内容 | 実装コスト | 効果 |
|---|---|---|---|
| A. 余白縮小 + バッジ追加 | 既存テーブルを維持しつつコンパクト化 | 低 | 中（横スクロール残存） |
| **B. 縦スタック型カードリスト** | 1頭=1行の横並び構造、flex-wrap で自然に折返し | 中 | **高** |
| C. 二段階展開 | タップで詳細展開、デフォルトは最小表示 | 高 | 高（実装複雑、UX 要検証） |

### 選定: **案B (縦スタック型カードリスト)**

**選定理由**:
- モバイル画面幅 (〜390px) で横スクロール不要
- `flex-wrap: wrap` により情報が自然に折返し、省スペース
- スコア順位を **左端の大型バッジ** で一目で把握
- **左端 5px の色帯 = EV 色** により EV ランクを瞬時に認識（緑/青/グレー）
- 右端に「EV 数値 + ラベル + スコア数値 + バー」を縦並び → **EV×スコア の対比が容易**
- 実装コストと効果のバランスが最良

二段階展開(C)は UX 検証が必要で過剰設計と判断。将来のダッシュボード追加時に必要ならその段階で導入。

## 3. デザイン (モバイル画面 〜390px)

```
┌──────────────────────────────────┐
│▮[1位] ⚫7  ◎エビスラッキー    EV 1.08🟢買い│
│▮       5.0倍(3人)・武豊・先・差  S 87 ▮▮▮▮▯│
├──────────────────────────────────┤
│▮[2位] 🔴3  ○サクセスホープ     EV 0.95   │
│▮       8.2倍(5人)・横山武・逃   S 72 ▮▮▮▯▯│
├──────────────────────────────────┤
│▮[3位] 🟡5  ▲ブライトスター ⚠️   EV 0.78   │
│▮       3.1倍(1人)・C.ルメ・先   S 68 ▮▮▯▯▯│
│         （左側の色帯がグレー = EV<1.00）      │
```

### 視覚階層の設計

1. **最も目立つ**: 左端 5px の EV 色帯 + スコア順位バッジ（2.3rem 幅）
2. **次に目立つ**: 馬名 + 予想印（色付き太字）+ 右端 EV バッジ（色×太字）
3. **補助情報**: オッズ・人気・騎手・脚質・調教 （0.7rem・グレー）

### 情報の左右配分

- **左**: 順位 → 枠色丸 → 馬名・印（識別情報）
- **右**: EV バッジ（色）+ スコア数値＋バー（定量情報）
- この配置で「EV が高い馬のスコア順位は？」「スコア1位の EV は？」が横一列で視認可

## 4. 実装詳細

### 追加した部品

| 名称 | 役割 |
|---|---|
| `evLabel(ev)` | EV値からラベル文字列（強気/買い/検討/-） |
| `WAKU_COLORS` | JRA 枠色 (CLAUDE.md と統一) |
| `RankBadge` | スコア順位の大型バッジ (1位=緑/2-3位=橙/その他=灰) |
| `HorseRankRow` | 1頭分の横並びカード行（左EV色帯＋順位＋枠馬番＋名＋EV＋スコア） |

### 置き換え箇所

- 旧: `<table style={{ minWidth: 600 }}>` with 9 columns × N rows
- 新: `<div>` container with `byScore.map((h, i) => <HorseRankRow ... rank={i+1} .../>)`
- タイトル: 「全頭データサマリー」→ **「全頭EV一覧」**
- 副題: 「総合スコア降順」→ **「スコア順位 × 期待値（EV色分け）」**

### レスポンシブ対応

- `flex-wrap: wrap` を使用、メディアクエリ不要
- 幅 390px 以下: 馬名行 + 右側メトリクスが縦に折返す
- デスクトップ (〜1000px+): 横一列に自然収納、破綻なし

### スコア順位の可視化

- `RankBadge` で「N位」を常に最左に配置
- 1位=緑背景／2-3位=橙背景／他=グレー背景 の 3 段階コントラスト
- 視線の導線が「順位→名前→EV→スコア」の自然な流れ

### EV 色分けの維持

- 既存 `evColor(ev)` 関数をそのまま使用（EV≥1.10=緑 / ≥1.00=青 / 未満=グレー）
- 新たに **左端 5px の色帯 (border-left)** で全行に適用
- 右上 EV バッジも同じ色でコーディネート → 左右で色が呼応

### 既存ロジックへの影響

- **`lib/score/calculator.ts`**: 変更なし
- **Section 5A (3段階推奨)**: 変更なし（要件通り）
- **Section 5 (馬券推奨 全券種)**: 変更なし
- 他セクション: 変更なし
- 既存のスコア・EV・推奨印の計算は一切触れていない

- モバイルで馬名が長い場合 `text-overflow: ellipsis` で省略される

- 枠色バッジの丸型は色覚バリエーションに対してコントラスト差で識別可能だが、  
  厳密な WCAG AA は今後の課題
