## Step 2: 2 つの原因を特定

### 原因1: 朝一レースは HTML に馬場状態が記載されない (125R)

netkeiba shutuba.html は「発走前のレース」時点ではまだ馬場状態が発表されていないため、HTML 上に `馬場:◯` の記載がない。

例: 202606030301 (2026-04-04 福島 1R, 発走 10:05)
```
<div class="RaceData01">
  10:05発走 / ダ1200m (右) / 天候:雨  ← 馬場記載なし
</div>
```

一方、同日の後続レース (4R〜12R) では馬場状態が公表されている。

### 原因2: 「稍」略記に scraper の正規表現が未対応 (12R)

2026-02-15 小倉1回8日目の全12R は、時刻に関係なく HTML に `馬場:稍` と略記されていた:

```
<span class="Item04">/ 馬場:稍</span>
```

scraper の正規表現 `/馬場[:：](良|稍重|重|不良)/` は **「稍」 (稍重の略)** にマッチしない。

---

## Step 3: 対応 (2 アプローチ)

### アプローチ1: 同日同場の最頻値で補完 (125R)

`scripts/fill_track_condition.ts`:
- raceId 先頭 10 文字 (年+場+回+日) でグループ化
- 同グループの既知馬場状態の最頻値を採用
- 同率の場合は 良 > 稍重 > 重 > 不良 の優先順

```
既に取得済み: 793 R
補完済み:     125 R ✓
補完不可:      12 R (同prefix全レース未設定 = 2026100108)
```

### アプローチ2: scraper 修正 + 12R 再スクレイプ

`lib/scraper/netkeiba.ts` の修正:
```diff
- const trackCondition = raceDataText.match(/馬場\s*[:：]\s*(良|稍重|重|不良)/)?.[1];
+ const trackRaw = raceDataText.match(/馬場\s*[:：]\s*(良|稍重|稍|重|不良)/)?.[1];
+ const trackCondition = trackRaw === '稍' ? '稍重' : trackRaw;
```

残り 12R を再スクレイプ → 全て「稍重」として取得成功。

## 補足知見

### 馬場状態ゼロ件の天候は?
「不明」137R の天候分布:
- 晴: 104 R / 曇: 19 R / 雨: 10 R / 小雨: 4 R

→ **晴天が多い** (発走前でも晴れなら馬場を想定しやすい)。天候と馬場状態は弱い相関。

### scraper 改修の横展開
「稍」「稍重」の表記ゆれは**競馬場により異なる可能性**があり、同様の表記ゆれが他の競馬場でも発生しうる。今回の `trackRaw === '稍' ? '稍重' : trackRaw` 変換は将来的に他の表記ゆれにも対応しやすい構造。

### 朝一レースの構造的欠損は今後も発生
1Rなど朝早いレースは HTML に馬場情報がないため、同日補完ロジック (fill_track_condition.ts) は **今後の再収集でも継続的に必要**。
