# スクリプト仕様

## スクリプト一覧

| ファイル                  | 用途                                       |
|---------------------------|--------------------------------------------|
| test-scraper.ts           | スクレイパーの動作確認                     |
| test-score.ts             | スコア・期待値計算の健全性チェック         |
| backtest.ts               | 過去レース結果でEV精度を検証               |
| optimize-correction.ts    | 補正係数（CORRECTION_FACTOR）の最適化      |

---

## test-score.ts の実行と期待出力

```bash
USE_MOCK=true pnpm tsx scripts/test-score.ts
```

### 合格基準（これを満たさない場合はロジックを見直す）
```
✓ 全馬スコア 0〜100: OK
✓ EV中央値 0.65〜0.85: 0.782
✓ 人気馬（〜10倍）平均EV: 0.94  ← 0.75〜1.05 の範囲内
✓ 人気薄（30倍超）平均EV: 0.64  ← 0.90 未満
✓ 最高EV が 1.5 を超えていない
```

## backtest.ts の仕様

### 入力
- 過去レース結果CSVファイル（`data/results/*.csv`）
- CSV列: `raceId, horseId, horseNum, odds, score, finished（着順）`

### 処理
1. 各馬のadjProbを計算
2. 実際の着順と突き合わせて的中判定
3. 券種ごとの回収率を集計

### 出力
```
=== バックテスト結果（過去Nレース）===
単勝   回収率: 82.3%  的中率: 24.1%
複勝   回収率: 88.7%  的中率: 51.3%
馬連   回収率: 76.4%  的中率: 12.8%
三連複 回収率: 71.2%  的中率:  6.4%
三連単 回収率: 68.9%  的中率:  2.1%

最適補正係数候補: 0.18（現在: 0.20）
```

// 補正係数を 0.05〜0.30 の範囲で0.01刻みでグリッドサーチ

## 実行時の注意事項

1. **スクリプトは本番データに書き込まない**
   → 読み取り専用で使うこと

2. **実ネットワーク使用時は必ず確認**
   ```bash
   # 実行前に必ず確認
   echo $USE_MOCK  # "false" になっていないか確認
   ```

3. **補正係数の変更は手動レビュー必須**
   → `optimize-correction.ts` の出力を見て、開発者が判断して変更する
   → クロコが自動で `CORRECTION_FACTOR` を書き換えることを禁止する

4. **結果CSVのコミット**
   → `data/results/` は `.gitignore` に含める（個人の分析データのため）

## バッチ処理実行時の注意事項

### collect-verification.ts 実行前チェック

```bash
# 必ず実行前に確認
curl -I "https://race.netkeiba.com/race/shutuba.html?race_id=202606030811"
# HTTP 200が返ることを確認してから実行
```

### 推奨される実行パターン
- 1日200レース以内
- 30分間隔で10分の休憩を自動挿入
- 夜間（2-6時）は実行禁止
- 実行前に前回の実行から24時間空ける

### エラー対応
- HTTP 400が出たら即座に中断
- 次回実行は最低3時間後
- 連続発生時は翌日以降に延期
